# 🎴 팔자 바이럴 자동화 시스템 - 구상안

## 목표
palja.net의 **띠별 주식운세** + **오행 온도계** 데이터를 활용,
매일 5개 채널(네이버블로그, 티스토리, 인스타, 스레드, 유튜브 숏츠)에
자동으로 컨텐츠를 생성하고 배포하는 시스템

---

## 시스템 아키텍처

```
[Supabase DB]
   │
   ├─ daily_zodiac_fortune (띠별 주식운세)
   ├─ five_element_ticker  (오행 온도계)
   └─ market_data          (시장 데이터)
            │
            ▼
   [OpenClaw Cron] ← 매일 07:00 / 09:00 / 14:00 등 스케줄
            │
            ├─ 1. DB 조회 → 오늘 데이터 로드
            ├─ 2. AI 컨텐츠 생성 (채널별 포맷 변환)
            ├─ 3. 이미지 생성 (인스타/숏츠용)
            └─ 4. 채널별 업로드
```

## 데이터 흐름

### 입력 (출처)
| 데이터 | 출처 | 설명 |
|--------|------|------|
| 띠별 운세 | palja.net DB | 12띠 × 오늘의 운세 + 추천주 |
| 오행 온도계 | palja.net DB | 오행(목화토금수) 기운 수치 |
| 시장 데이터 | 외부 API | 코스피 등락, 업종별 동향 |

### 출력 (5개 채널)
| 채널 | 컨텐츠 형태 | 생성 방식 |
|------|------------|----------|
| **네이버 블로그** | SEO 친화적 장문 글 | 1000자 내외, 키워드 최적화 |
| **티스토리** | 네이버와 유사 + 추가 태그 | 1000자 내외, 복붙 최적화 |
| **인스타그램** | 이미지 + 짧은 캡션 | 카드뉴스 형태, 5개 해시태그 |
| **스레드** | 가벼운 텍스트 스레드 | 3-5줄, 밈 감성 톤 |
| **유튜브 숏츠** | 15~30초 영상 (음성+텍스트) | AI TTS + 이미지 슬라이드 |

## 컨텐츠 아이디어

### 1️⃣ 매일 아침 띠별 주식운세 (07:00)
**원천:** DB daily_zodiac_fortune
**컨셉:** "오늘 ○○띠님, 주식운세 대박/쪽박"

**채널별 버전:**
- **블로그:** "7월 14일 띠별 주식운세 완벽 정리! 쥐띠 대박, 돼지띠 조심"
- **인스타:** 깔끔한 카드뉴스 12띠 한눈에 (1-2페이지)
- **스레드:** "오늘 내 띠 주식운세 봤냐? 🤯" 한줄 요약
- **숏츠:** "오늘 상승할 띠 top3 vs 하락할 띠 top3"

### 2️⃣ 오행 온도계 리포트 (09:00 개장 전)
**원천:** DB five_element_ticker
**컨셉:** "오늘 시장은 불기운이 강하다" → 추천 업종

**예시:**
```
🔥 오행 온도계: 火(화) 82% / 木(목) 45% / 土(토) 33%
→ 오늘은 화(火) 기운! 전기/전자/2차전지 주목!
→ AI 추천: 삼성SDI, LG에너지솔루션 등
```

### 3️⃣ 긴급 트리거 (시장 급변 시)
**원천:** 시장 데이터 급등/급락 감지
**컨셉:** "오늘 폭락! 내 팔자로 보는 대응법"

**예시 (오늘처럼):**
```
🚨 코스피 -8% 서킷브레이커!
팔자가 보는 오늘의 운세:
→ 오늘 태어난 ○○띠는 보유, ××띠는 손절 타이밍
→ 오행: 金(금) 기운 강한 날 = 반도체株 조정
```

### 4️⃣ 주간/월간 결산 (일요일)
**원천:** 누적 데이터
**컨셉:** "이번주 띠별 운세 적중률 체크"

## 기술 스택 (제안)

| 역할 | 도구 |
|------|------|
| **스케줄러** | OpenClaw Cron |
| **DB** | Supabase (기존) |
| **AI 컨텐츠 생성** | OpenClaw agent (제가 직접) |
| **이미지 생성** | 추후 결정 (별도 도구 필요) |
| **블로그 업로드** | 네이버 API / 티스토리 API |
| **인스타 업로드** | Instagram Graph API |
| **숏츠** | YouTube Data API v3 |
| **환경변수** | 로컬 .env (gitignored) |

---

## 실행 순서

### Phase 1 - 컨텐츠 생성 자동화 ✅
1. Supabase 연결 → 데이터 조회
2. AI가 5개 채널별 컨텐츠 한번에 생성
3. 생성 결과를 로컬 JSON/마크다운으로 저장
4. **= CEO님이 직접 복붙해서 올리기**

### Phase 2 - 배포 자동화
5. 각 채널 API 연동
6. 자동 업로드 스크립트
7. 일정에 따라 발행

### Phase 3 - 트리거 고도화
8. 시장 급변 감지
9. 자동 컨텐츠 생성 + 배포
10. 성과 측정 및 최적화
